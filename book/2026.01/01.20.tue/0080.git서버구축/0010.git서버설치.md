자체적인 Git 서버를 구축하는 방법은 크게 Bare 저장소를 이용한 수동 구축과 Gitea와 같은 GUI 관리 도구를 사용하는 방법이 있습니다. 여기서는 실무에서 가장 많이 활용되는 **Gitea**를 이용한 서버 구축과 이를 활용한 기초 사용법을 단계별로 다룹니다.

---
####0 깃 지우기
```
sudo apt remove git
sudo pkill -u git 

```

### 1. Git 서버(Gitea) 설치 및 환경 구성

#### 1.1 시스템 업데이트 및 Git 설치

서버의 패키지 목록을 갱신하고 Git 코어 패키지를 설치합니다.

```bash
sudo apt update
sudo apt install -y git sqlite3 curl

```

#### 1.2 Gitea 패키지 설치

`apt` 저장소를 추가하여 Gitea를 설치합니다.

```bash
# 저장소 스크립트 실행
curl -s https://packagecloud.io/install/repositories/gitea/gitea/script.deb.sh | sudo bash

# Gitea 설치
sudo apt install -y gitea

```

#### 1.3 서비스 활성화 및 포트 개방

```bash
# 서비스 시작 및 자동 실행 설정
sudo systemctl enable --now gitea

# 방화벽 허용 (기본 포트: 3000)
sudo ufw allow 3000/tcp

```

> **AWS 과금 주의**: EC2 인스턴스를 사용하는 경우, 보안 그룹(Security Group)에서 **3000번 포트** 인바운드 규칙을 반드시 추가해야 합니다. 인스턴스 실행 시간에 따른 비용과 데이터 전송 비용이 발생할 수 있습니다.

---

### 2. Git 서버 초기 설정 (Web GUI)

서버 설치 후 브라우저를 통해 초기 설정을 완료해야 합니다.

1. **웹 접속**: `http://서버_IP:3000`으로 접속합니다.
2. **데이터베이스 설정**: `SQLite3`를 선택하면 별도의 DB 설치 없이 즉시 사용 가능합니다.
3. **서버 도메인/URL**: 서버의 IP 주소 또는 도메인을 정확히 입력합니다. (예: `http://192.168.1.100:3000`)
4. **관리자 계정**: 최하단의 관리자 계정 설정 메뉴에서 사용할 아이디와 비밀번호를 생성합니다.

---

### 3. Git 기초 사용 방법 (Client Side)

서버 구축이 완료되었다면, 로컬 PC(사용자) 환경에서 서버를 활용하는 기본 흐름을 익힙니다.

#### 3.1 사용자 정보 등록

커밋 로그에 남을 이름과 이메일을 설정합니다.

```bash
git config --global user.name "YourName"
git config --global user.email "uvin@naver.com"

```

#### 3.2 원격 저장소 복제 (Clone)

서버에 생성된 빈 저장소를 로컬로 가져옵니다.

```bash
git clone http://서버_IP:3000/사용자명/저장소명.git
cd 저장소명

```

#### 3.3 파일 생성 및 스테이징 (Add)

새로운 파일을 생성하고 Git의 추적 대상에 포함시킵니다.

```bash
echo "# Project Title" > README.md
git add README.md

```

#### 3.4 로컬 저장소 기록 (Commit)

수정된 내용을 로컬 저장소에 영구적으로 기록합니다.

```bash
git commit -m "Initial commit: Create README"

```

#### 3.5 서버로 전송 (Push)

로컬의 커밋 내역을 서버(Gitea)로 업로드합니다.

```bash
git push origin main

```

---

### 4. 핵심 작업 흐름 요약표

| 단계 | 명령어 | 설명 |
| --- | --- | --- |
| **복제** | `git clone [URL]` | 서버의 프로젝트를 로컬로 가져오기 |
| **확인** | `git status` | 현재 파일들의 상태 확인 |
| **준비** | `git add [파일명]` | 커밋할 파일 선택 (Staging) |
| **기록** | `git commit -m "메시지"` | 로컬 저장소에 변경사항 저장 |
| **동기화** | `git pull origin main` | 서버의 최신 변경사항 가져오기 |
| **업로드** | `git push origin main` | 로컬 작업 내용을 서버에 반영 |
