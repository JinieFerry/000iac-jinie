## GitHub 연동 및 원격 저장소 관리

로컬 환경(내 컴퓨터)에서 Git으로 관리하던 프로젝트를 GitHub와 연동하면, 코드의 안정적인 백업은 물론 다른 개발자와의 협업 및 AWS와 같은 클라우드 서비스로의 자동 배포(CI/CD)가 가능해집니다.

---

### 1. GitHub 연동의 핵심 개념

GitHub 연동은 내 컴퓨터의 **Local Repository**와 GitHub 서버의 **Remote Repository**를 연결하는 과정입니다.

* **Push**: 로컬의 변경 사항을 원격 저장소(GitHub)로 업로드합니다.
* **Pull**: 원격 저장소의 변경 사항을 로컬로 가져와 업데이트합니다.
* **Fetch**: 원격 저장소의 변경 이력만 확인하고 로컬 코드와 합치지는 않습니다.

---

### 2. 단계별 연동 방법 (CLI 기준)

#### 2.1. GitHub 저장소 생성

1. GitHub 로그인 후 오른쪽 상단의 **+** 버튼 -> **New repository**를 클릭합니다.
2. **Repository name**을 입력하고 'Create repository'를 누릅니다.

#### 2.2. 로컬과 원격 연결하기

터미널이나 VS Code 내장 터미널에서 다음 명령어를 순서대로 입력합니다.

```bash
# 1. 로컬 저장소 초기화 (이미 되어 있다면 생략)
git init

# 2. 파일 스테이징 및 첫 커밋
git add .
git commit -m "First commit"

# 3. 원격 저장소 주소 등록 (origin은 별칭임)
git remote add origin https://github.com/사용자이름/저장소이름.git

# 4. 브랜치 이름을 main으로 변경 (최신 표준)
git branch -M main

# 5. 로컬 코드를 GitHub로 업로드
git push -u origin main

```

### 3. 인증 방식 선택 (HTTPS vs SSH)

GitHub는 보안을 위해 2021년부터 일반 비밀번호 로그인을 통한 Push를 제한하고 있습니다.

| 방식 | 특징 | 권장 상황 |
| --- | --- | --- |
| **HTTPS (PAT)** | **Personal Access Token**을 생성하여 비밀번호 대신 사용합니다. | 초기 설정이 간편하여 입문자에게 권장됩니다. |
| **SSH** | 내 컴퓨터의 공개키(Public Key)를 GitHub에 등록합니다. | 매번 인증할 필요가 없어 실무 및 자동화 환경에 적합합니다. |

---

### 4. AWS 연동 및 비용/보안 가이드

GitHub에 코드를 올린 후 AWS 인프라와 연결할 때 반드시 확인해야 할 사항입니다.

#### 4.1. 보안 사고 방지 (중요)

* **자격 증명 유출**: AWS IAM Access Key가 포함된 파일이 GitHub Public 저장소에 올라갈 경우, 수 분 내에 해커들이 이를 가로채 대규모 채굴 인스턴스를 생성할 수 있습니다.
* **해결책**: `.gitignore` 파일에 비밀번호나 키가 포함된 설정 파일을 반드시 등록하십시오. 이미 올라갔다면 즉시 해당 Key를 AWS 콘솔에서 비활성화해야 합니다.

#### 4.2. AWS 관련 유료 서비스 주의

* **AWS Amplify / App Runner**: GitHub와 연결하여 자동 배포를 설정할 수 있습니다.
* **과금**: 빌드가 실행될 때마다 빌드 시간에 따른 요금이 발생하며, 호스팅되는 데이터 전송량에 따라 비용이 청구됩니다.


* **AWS CodeStar Connections**: GitHub와 AWS 서비스를 안전하게 연결해주는 통로 역할을 하며, 연결 자체는 무료이나 이를 통한 데이터 처리 비용은 별도로 발생할 수 있습니다.
